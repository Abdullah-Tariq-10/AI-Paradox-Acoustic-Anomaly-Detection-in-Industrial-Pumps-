# Acoustic Anomaly Detection in Industrial Pumps

This repository contains an end-to-end Machine Learning pipeline designed for **Industrial Acoustic Anomaly Detection**. Developed for the *AI Paradox* competition, the system analyzes 10-second sound recordings of industrial pumps to automatically distinguish between normal operations and subtle acoustic deviations indicative of machine faults or structural abnormalities.

By integrating comprehensive time- and frequency-domain audio feature extraction with a multi-model **Stacked Ensemble Framework**, the architecture achieves near-perfect classification performance.

## 📊 Project Performance Summary

* **ROC-AUC Score:** `0.9987` 


* **Overall Accuracy:** `99.0%` 


* **Macro F1-Score:** `0.98` 


* **Abnormal Recall:** `0.96` (Crucial for minimizing undetected machine faults)

## 🛠️ System Architecture & Pipeline

```
[ Raw WAV Audio (22,050 Hz) ] 
            │
            ▼
[ Preprocessing & Data Augmentation ] (Noise, Time Shift, Pitch Shift)
            │
            ▼
[ Comprehensive Feature Extraction ] (MFCCs, Chroma, Spectral Contrast, etc. -> 233 Dim)
            │
            ▼
[ Preprocessing Scaling & Clipping ] (RobustScaler + Outlier Clipping)
            │
            ▼
[ 5-Fold Stratified Cross-Validation ]
   ├── LightGBM ───────┐
   ├── CatBoost ───────┼──> [ OOF Predictions / Meta-Features ]
   ├── Random Forest ──┤                  │
   └── GradientBoost ──┘                  ▼
                               [ Meta-Learner: Logistic Regression ]
                                          │
                                          ▼
                               [ Optimized Thresholding (0.550) ] ──> [ Final Prediction ]

```

### 1. Data Preprocessing & Augmentation

* **Standardization:** Audio tracks are systematically resampled to **22,050 Hz**. Short tracks are padded, and long clips are uniformly trimmed to prevent dimensionality mismatch.


* **Robustness Augmentation:** To combat a native class imbalance (**914 Normal vs. 330 Abnormal** samples after generation) and boost model generalization, a dynamic data augmentation pass randomly applies:


* *White Noise Injection* (simulates industrial factory floors) 


* *Time Shifting* (rolls audio array to simulate varying capture offsets)


* *Pitch Shifting* (alters tonal qualities without breaking alignment) 


### 2. High-Dimensional Acoustic Feature Engineering

A 233-dimensional feature vector is extracted from each audio time series utilizing `librosa` and `scipy.stats`:

* **Mel-Frequency Cepstral Coefficients (MFCCs):** Mean, standard deviation, minimum, and maximum of the top 20 coefficients (80 features).


* **Spectral Descriptors:** Spectral Centroid, Spectral Rolloff, Spectral Bandwidth, and **Spectral Contrast** to identify frequency distributions and peak/valley discrepancies.


* **Chroma STFT & Tonnetz:** Captures harmonic, tonal, and pitch transitions across the clips.


* **Mel Spectrogram Energy:** 40 Mel-scaled frequency bands converted to decibel format.


* **Temporal & Statistical Metrics:** Root Mean Square (RMS) energy, Zero-Crossing Rate (ZCR), skewness, and kurtosis to extract structural amplitude profiles.



### 3. Model Pipeline & Stacking Strategy

The engine utilizes a two-tier **Stacked Ensemble Architecture** trained via a **5-Fold Stratified Cross-Validation** routine to guarantee complete out-of-fold validation transparency:

* **Base Classifiers (Layer 0):** Highly tuned instances of **LightGBM**, **CatBoost**, **Random Forest**, and **Gradient Boosting**. Class imbalance is handled natively in this layer using automated inverse class weighting ratios (e.g., `class_weight='balanced'`).


* **Meta-Learner (Layer 1):** A regularized **Logistic Regression** classifier that ingests the out-of-fold probability distributions generated by the base estimators to output the optimal decision boundary.


* **Threshold Tuning:** Instead of settling for a default 0.5 partition, a precision-recall sweep determined that a threshold of **`0.550`** maximizes minority class F1-performance while keeping false alarms to a minimum.

## 🚀 How to Reproduce Natively

### 1. Prerequisites

Clone this repository and verify your python environment (`>= 3.10`). Install the required dependencies:

```bash
pip install numpy pandas librosa scikit-learn lightgbm catboost tqdm matplotlib scipy

```

*Note: Your operating system may require `libsndfile` installed globally to allow `librosa` to process raw `.wav` extensions.*

### 2. Configuration Setup

Ensure your local data paths reflect the directory mappings used inside the script:

```python
TRAIN_NORMAL = "path/to/train/normal"
TRAIN_ABNORMAL = "path/to/train/abnormal"
TEST_PATH = "path/to/test"

```

### 3. Execution

Run the Jupyter notebook cells or export it directly as a python script to trigger features compilation, ensemble synchronization, and test predictions generation:

```bash
python ai-paradox.py

```


## 🚀 Production Deployment & Future Enhancements

To transition this framework from a batch-inference research notebook to an enterprise-grade, real-time predictive maintenance asset, the following production strategies are proposed:

### 1. Edge Hardware Optimization (NVIDIA Jetson Nano)

Running deep tree ensembles and high-dimensional audio feature extraction can introduce memory and processing overhead on lightweight edge devices. Future milestones include:

* **Feature Pruning:** Utilizing variance thresholding and permutation feature importance to reduce the 233-dimensional feature vector to a lean set of under 50 critical features, drastically saving edge CPU compute cycles.
* **Model Quantization:** Exporting base models to an ONNX (Open Neural Network Exchange) format to leverage hardware acceleration natively on low-latency edge computing devices deployed right next to the physical industrial pumps.

### 2. Stream-Based Real-Time Inference Architecture

Instead of processing static 10-second blocks, a real-time production deployment can utilize a sliding window queue approach:

```
[ Industrial Microphone / Sensor ]
               │ (Continuous Audio Stream)
               ▼
   [ PyAudio / SoundDevice Stream ]
               │ (Chunking: e.g., 1024-sample frames)
               ▼
    [ Rolling Ring Buffer Queue ] ──> Maintains latest 10-second window
               │ (Every 1-2 seconds)
               ▼
[ Fast Feature Extraction Pipeline ] (Librosa Optimized Vectorized Pass)
               │
               ▼
  [ ONNX Ensemble Prediction ]
               │
               ▼
 [ Target Probability > 0.550? ] ──(Yes)──> [ Trigger Warning Alert / API Webhook ]

```

### 3. Lightweight Microservice Deployment (FastAPI)

For centralized cloud or on-premise localized server setups, the ensemble model can be wrapped in a high-performance asynchronous API framework (`FastAPI`).

* **Endpoint:** `POST /api/v1/predict-anomaly`
* **Payload:** Raw binary audio stream or `.wav` multipart file.
* **Response Latency target:** `< 150ms` (inclusive of downsampling, calculation, and model execution).
* **Alert Integration:** Connects downstream to operational notification systems (e.g., Slack Webhooks, PagerDuty, or MQTT brokers for industrial SCADA integration) when probability thresholds surpass the optimized **0.550** barrier.
